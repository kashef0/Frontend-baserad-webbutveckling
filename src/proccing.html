Arbetsprocess för Utveckling och Publicering

Automatisering & publicering

Välkommen till min sida där jag berättar om hur min process går till för att utveckla och publicera webbplatser. Målet är att göra en webbplats automatiska för att öka effektiviteten i arbetsprocess och förbättra kvaliteten på det slutliga resultatet.

Verktyg och Tekniker:

  • Byggverktyg: Parcel:
    Jag föredrar att använda Parcel som mitt främsta byggverktyg. Parcel är en snabb och enkel pakethanterare som automatiserar hanteringen av HTML, CSS, JavaScript och bilder utan att behöva konfigurera något.
  • Automatisering av CSS:
    jag skapade flera css filer till exempel: en css-fil för mobilnav, navmenu och logotyp. det gör enklare att fixa eller ändra css filen senare.
    Minimering av CSS och konvertering av SCSS/SASS till CSS hanteras automatiskt genom användning av Parcel och detta säkerställer att css filer är optimerade och laddas snabbt. men i den här momenten kommer vi bara att minifera css filer.
  • Automatisering av JavaScript:
    jag skapade två-js filer en för mobilemenu och den andra för klockan och sedan import den andra js-filen till den första. Efter det JavaScript-koden omvandlas och komprimeras automatiskt med hjälp av Parcel.
    Det innebär att jag kan använda den senaste JavaScript-syntaxen samtidigt som jag säkerställer att min kod fungerar på äldre webbläsare.
  • Automatisering av HTML:
    jag skapade tre olika html-filer och minifiering av HTML görs automatiskt när jag bygger med Parcel. Detta leder till en optimerad HTML-kod.
  • Optimering av Bilder: Jag använder "sharp-paket" för att automatiskt optimera och komprimera bilder när de uppdateras. Dessutom konverterar jag bilderna till moderna format som WebP eller AVIF för att förbättra prestandan samt jag skapade en sharp. config.json fil för att manuellt lägga till inställningar till exempel ("webp": "quality": 75,) och det är kvaliteten på bild och använde picture-tag med source som i det exempel: ( "srcset="../src/images/helsingborg.jpg?as=webp&width=800" type="image/webp") och det betyder att bilden kommer att konvertera från jpg till webp format med bredd 800.
  • Bakåtkompatibilitet:
    Genom att använda Parcel och deklarera vilka webbläsare som stöds av min app med hjälp av webbläsarlist i min package.json ser jag till att JavaScript-filerna är kompatibla bakåt, så att webbplatsen kan fungera i så många webbläsare som möjligt.

arbetsprocess

Jag började med att skapa ett repo på GitHub och sedan kopierade jag repositoriet från GitHub till min dator. Sedan körde jag npm init -y för att skapa ett npm-projekt och fylla i standardinformationen. Därefter installerade jag alla projektberoenden som finns i package.json och andra relaterade filer i mitt projekt genom att använda kommandot npm install parcel --save-dev.

Efter det skapade jag en fil med namnet .gitignore eftersom jag inte ville inkludera alla paket i versionshanteringen. Jag angav i den här filen (node_modules/) för att ignorera det paketet och fokusera på de viktiga filerna. Jag skapade också en ny mapp och kallade den src. I den här mappen lade jag till fyra undermappar (css, js, pictures, log-icon) och tre HTML-filer (index.html, picture.html, proccing.html).

I package.json ändrade jag några inställningar, som source där jag lade till sökvägen till min startsida (src/index.html). I package.json ändrade jag även några inställningar för skript (start: parcel för att köra webbplatsen i live-server, build: "parcel build" för att paketera ihop webbplatserna och göra dem redo för publicering). Slutligen körde jag npm run build och publicerade min webbplats på Netlify.
